论文解读: 2017 Transformer

|Word count:2.1k|Reading time:7min|Post View:

论文地址：[1706.03762] Attention Is All You Need (arxiv.org)

论文讲解：Transformer论文逐段精读【论文精读】_哔哩哔哩_bilibili

Motivation

主流的序列转录模型主要基于循环神经网络或卷积神经网络，这些模型通常包含encoder和decoder。在这些性能好的模型中往往使用注意力机制来连接 encoder 和 decoder。但循环神经网络生成当前时刻的隐藏状态 ht 需要使用前一个时间的隐藏状态 ht-1 和当前时刻t的输入，因为这样的一种顺序性，所以并行能力很差，而且每次都要记住前面的隐藏状态，所以对内存要求也高。所以作者提出了 Transformer，这是一个纯注意力机制的网络，所以并行度高，而且这可以称为 MLP，CNN 和 RNN 以外的第四类基础模型。作者预测 Transformer 在将来也能运用在图像等领域，当然这几年 Transformer 在这些领域的确取得了巨大的成功。

Background

有一些工作是用卷积神经网络来代替循环神经网络，以此来减少时序的计算，提高并行度。但是如果两个像素距离很远的话，需要多层卷积才能将两个像素进行融合。所以对于距离远的两个位置，学习依赖性是比较困难的。但是注意力可以直接一次看见所有的像素。卷积也有个优点，就是可以做多个输出通道，不同的输出通道可以认为是识别不同的模式，所以 Transformer 中使用了 Multi-Head Attention（多头注意力），来模拟多个输出通道。

当时 Transformer 是第一个只用自注意力机制来做 encoder-decoder 的模型。

Model Architecture

Transformer 就是一个 encoder-decoder 的结构，左边是编码器，右边是解码器，先看看它的框架图：

Encoder and Decoder Stacks

Encoder: 编码器中有 N=6 个相同的层（layer），那么这个其实和 ResNet 中的 block 差不多。每个 layer 有2个子层（sub-layer），第一个子层是多头自注意力机制（multi-head self-attention mechanism），第二个就是全连接层（position-wise feed-forward network）。在图中可以看到，两个子层后面都跟了一个 Add & Norm，而且左边有黑色的连接线（下图中红色框内），这是残差连接：

那么子层的输出根据 Add & Norm 和残差连接就可以表示为 LayerNorm(x+Sublayer(x))，为了方便残差连接，每个子层和嵌入层的输出都为 \(d_{model}=512\)

那么什么是 LayerNorm？BatchNorm 是把每个特征进行均值为0，方差为1的标准化，而 LayerNorm 则是把每个样本进行均值为0，方差为1的标准化。为什么要用 LayerNorm？因为在序列模型中，每个序列的长度是不一样的，所以如果在预测的时候遇到一个过长的样本，那么 BatchNorm 可能不会很好用，因为在训练样本中没有见过这么长的样本，但是 LayerNorm 就没有这种问题，因为是样本本身做的标准化，也不需要像 BatchNorm 那样记录全局的均值和方差。

Decoder: 解码器和编码器一样也有6个层，但是子层有3个。因为注意力机制会看到整个序列，但是解码器在预测第 t 时刻输出的时候不应该看到 t 时刻之后的输入，所以用了 Masked Multi-Head Attention，来保证训练和预测的行为是一致的。

Attention

注意力机制的简介，不做介绍

下图是论文中使用的注意力机制的图：

Scaled Dot-Product Attention

其实这个注意力和 Dot-Product Attention 差不多，就多了 Scale 和 Mask。因为 query，key 和 value 都有好多个，所以为了提高运算效率，采用了矩阵运算。首先 Q 矩阵由 query 组成，K 矩阵由 key 组成， query 和 key 的维度都为 \(d_k\)，V 由 value 组成，value 的维度为 \(d_v\)。

Scaled Dot-Product Attention 的公式： \[ \begin{equation} Attenion\left(Q, K, V\right)=softmax\left(\dfrac{QK^T}{\sqrt{d_k}}\right)V \end{equation} \] 最找 Figure 2 中的左图，首先是 Q 和 KT 进行矩阵相乘，这么做的目的是求出每个 query 和 key 的内积，即输出矩阵的每一行的每一列分别是该 query 和所有 key 的内积值：

实际上 Q，K，V 三个矩阵的维度是一样的，因为 Transformer 使用的是自注意力机制，所以可以把式子理解为： \[ \begin{equation} Attenion\left(X, X, X\right)=softmax\left(\dfrac{XX^T}{\sqrt{d_x}}\right)X \end{equation} \]

接着就是 Scale，Mask（opt.）和 softmax。其中 softmax 的作用就是计算出权重，即对每一行进行 softmax 计算。那么为什么会多一步 Scale，也就是除以 \(\sqrt{d_k}\)，当 \(d_k\) 不大的时候，除不除也没有太大的关系，但是如果 \(d_k\) 太大的话，这个内积的值会有可能很大，或者很小，那么在做 softmax 操作的时候会导致部分值过于靠近1或0，在算梯度的时候，梯度太小。所以做 Scale 使方差变小，防止内积太极端。

Mask（opt.）说明 Mask 操作是选择性的，从框架图中可以看出只有在编码器中才需要 Mask 操作。因为注意力机制会一次性看到所有序列，但是在进行预测的时候，我们只希望在 t 时刻看到 K1 到 Kt-1 ，使用 Mask 的话会在 Q 和 K 相乘后，将 Kt 以及之后的值都赋值为特别的负数，比如 -110，就会使它们在 softmax 运算时都贴近于0。

最后就是将上面的输出结果和 Q 进行相乘，即得到加权和。

Multi-Head Attention

Figure 2 中的右图就展示了 Multi-Head Attention。因为卷积神经网络一个特点是能做多个通道，即每次用卷积去提取特征的时候会用多个卷积，多少个卷积就会得到多少个输出通道。每个通道其实是识别不同的模式。所以作者也使用了这种思想，将 Q，K，V 都先经过线性层投影到低纬度，然后计算注意力，投影和计算要运算 \(h=8\) 次，这么做就和多个卷积一样，然后将 h 次的输出结果合并，再做一次线性变换。 \[ \begin{equation} \begin{aligned} MultiHead \left(Q, K, V \right) &= Concat \left(head_1,..., head_h \right)W^O\\ where \text{ } head_i &= Attention \left (QW_i^Q, KW_i^K, VW_I^V \right) \end{aligned} \end{equation} \]

其中 \(W_i^Q \in \mathbb{R} ^ {d_{model} \times d_k}\)，\(W_i^K \in \mathbb{R} ^ {d_{model} \times d_k}\)，\(W_i^V \in \mathbb{R} ^ {d_{model} \times d_v}\)，\(W_i^O \in \mathbb{R} ^ {hd_v \times d_{model}}\)

因为 \(h=8\)，所以令 \(d_k = d_v = d_{model} / h = 64\)，这样在做 Scaled Dot-Product Attention 时，矩阵的参数数量和只有一个注意力层时是相同的。

Applications of Attention in our Model

因为使用的是自注意力机制，所以每次进行计算时 Q，K，V 三个都是完全一样的东西，另外从编码器传到解码器的时 K 和 V。

Position-wise Feed-Forward Networks

图中可以看到 Feed Forward，这其实就是一个但隐藏层的 MLP，公式为： \[ \begin{equation} FFN \left(x\right) = max \left(0, xW_1 + b_1\right) W_2 + b_2 \end{equation} \] 一个序列中，每一个词就是一个 position，那么对每个 position 都做同样的 MLP，这个前向传播网络的输入和输出 \(d_{model}=512\)，但是中间隐藏层输出的维度为 \(d_{ff}=2048\)，也就是说先把 512 映射到 2048，然后又映射回 512.

Positional Encoding

attention 没有时序信息，所以在词嵌入上加入位置编码： \[ \begin{align} PE_{(pos,2i)} = sin(pos/10000^{2i/d_{model}})\\ PE_{(pos,2i)} = sin(pos/10000^{2i/d_{model}}) \end{align} \]

Label Smoothing

在训练时，正常情况下 softmax 希望正的为1，负的为0，因为1太极端，一般会把1设为0.9，但 transformer 直接把1设定为0.1，这虽然会使 perplexity 提高，使模型不确信，但是会提高准确率和BLEU。
