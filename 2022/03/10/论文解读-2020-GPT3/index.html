论文解读: 2020 GPT3

Created2022-03-10|Updated2022-03-26|AI
|Word count:1k|Reading time:3min|Post View:

论文地址：2005.14165.pdf (arxiv.org)

论文解读：GPT，GPT-2，GPT-3 论文精读【论文精读】_哔哩哔哩_bilibili

简介

GPT2 虽然有创意，但有效性太低，GPT3 为了提高有效性，做了 few-shot。GPT3 模型非常的大，可学习参数有1720 亿个。

近年来，NLP 领域都是用预训练模型进行微调，但这还是有问题存在，需要下游任务的数据集，并且在下游任务还需进行微调。移除这两者的限制是有意义的：

  1. 每个下游任务都需要去收集相关的大量的数据，费时费力。
  2. 微调性能好不能代表预训练模型泛化性能好，可能是因为下游任务的数据在预训练的时候有重复的出现，数据分布太狭窄，导致微调的时候过拟合。比如用 Wikipedia 或网上爬的数据来做预训练，可能这些数据里已经包含了下游任务的数据或类似的数据，所以微调效果会比较好，但若换个语种或更专业的文本，可能泛化性会表现得不好。所以如果不做微调，那就能知道模型的泛化性到底好不好。
  3. 对于人类而言，对于学习一个语言任务，比如判断一个句子描述的是开心还是悲伤，不需要大量的有监督的数据，而且在一定程度上，人类在面对新的任务上时，也能有一个比较好的表现。所以作者希望语言模型也能够像人类一样，在多任务之间流畅高效地衔接。

其中一种可能的解决方法是 meta-learning 元学习，meta-learning 就是使模型在训练时去学习不同的任务，那么就能够学习到不同的模式识别能力，在进行推测的时候可以使用这些能力快速地适应任务。

而 GPT3 也用了 zero-shot，ont-shot，few-shot。in-context learning 就是说对于每个任务，哪怕是 one-shot 还是 few-shot，也不会再更新权重。

GPT3 是在三种情况下评估的：

  1. few-shot learning，对每个子任务提供 10 到 100 个训练样本
  2. one-shot learning
  3. zero-shot learning

下面两张图是三种情况的对比：

作者也解释了三种情况是什么样的：

Figure 2.1 中是以英文翻译为法语为例。右边的是微调，微调就是每次给下游任务的数据时，都会做梯度更新，但是 GPT3 是没有做微调的。zero-shot 的话，每次都会给一个自然语言描述 “Translate English to French” 来告诉模型我想做什么，然后给出 prompt，也就是希望被翻译的英文单词。one-shot 是在 task description 和 prompt 中间插入一条样本，虽然是样本，但模型不会算梯度来进行更新。这条样本是希望模型使用注意力机制，从中间抽取有用的信息，来帮助后面的英文翻译。所以叫做 in-context learning，上下文学习。few-shot 就是给的样本再多一些，但样本也不能过多，因为模型可能不能处理太长的序列。

虽然上面的做法对于新任务比较友好，但是若有很多数据，无法将这些数据都放进去做为 example，毕竟模型无法处理太长的序列。另外如果一个数据集在做 one-shot 的时候效果很差，做 few-shot 的时候效果很好，那么每次在做预测的时候都需要将数据放进去，模型没有记忆能力，不能记住这些样本中提取出来的有用的信息。

数据集

Common Crawl 数据集比较脏，所以用 GPT2 的数据集作为正例，Common Crawl 作为负例，进行二分类训练，然后再对 Common Crawl 进行预测，若为正例则保留。另外再对 Common Crawl 进行去重。将 BERT，GPT2，GPT 用的高质量数据集都拿过来补充。
