把本地的个人hexo博客部署到GitHub上

创建repository

创建新的Repository，输入Repository name，规则为 xxx.github.io，xxx为自己的用户名，一定要和用户名一样，其他可以不勾选，可以勾了一个Add a README file

在git bash中配置Github用户信息

输入上面两条命令，用户名和邮箱都是GitHub的用户名和邮箱

添加Deploy Keys

输入命令生成SSH,然后在git bash给出的路径“C:.ssh”下找到id_rsa.pub文件，用记事本打开，复制里面所有的内容

去刚刚生成的repository(xxx.github.io)中，添加 deploy key，Title自己设置，Allow write access不用打勾，将刚刚复制的内容粘贴到Key框框中，然后点击Add key添加完成

进行GitHub部署

继续在git bash中输入

hexo d经常因为网络原因不能一次完成，可能要重复多次才能成功

在第一次部署时会跳出一个信息，让你进行GitHub账号的绑定，正常操作即可，下面是两张图片，分别为上传失败，和上传成功，失败的话可能要重复几次，或者去网上看看有什么有用的方法。

设置Pages

回到之前创建的repository中，在Pages选项中将branch选为master

前面说了会考，因为前面设置的就是master，所以在进行部署时，会把文件上传的repository中的master中，而不是main中

查看个人博客

完成上面步骤后就可以去GitHub用户名.github,io网页查看自己的个人博客了

以后写博客的大概步骤就是： 1. 创建博客文件hexo new "xxx" 2. 用markdown来写该博客 3. 依次使用hexo cl,hexo g, hexo d来完成部署，但是一般部署后马上去自己的网页 GitHub用户名.github.io 的话会发现自己的网页并没有更新，可能需要等几分钟才会更新
