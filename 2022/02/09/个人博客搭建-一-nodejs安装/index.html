个人博客搭建(一): nodejs安装

Created2022-02-09|Updated2022-03-28|blog
|Word count:629|Reading time:2min|Post View:

下载nodejs安装包

nodejs安装

选择安装路径，然后Next

这部分默认全部选择即可，Next

就用来搭建个人博客，不用勾上，否则会安装很多东西，时间特别久，Next，等待安装完成

打开cmd，输入node -v，查看版本号，验证是否安装成功

更改模块安装路径和缓存路径，现在nodejs安装路径下创建node_cache文件夹，然后在终端输入 npm config set prefix "D:\nodejs" 和 npm config set cache "D:\nodejs\nodejs_cache" 来更改模块安装路径和缓存路径（具体路径按照自己的），否则会按照默认路径安装在C盘下，另外可以通过 npm config get prefix 和 npm config get cache 来查看设置的路径

配置环境变量

打开环境变量后，查看用户变量的Path，如果有这样一条变量，请删除

点击下方系统变量的新建，注意是系统变量不是用户变量，变量名为Node_PATH，变量值为node_modules（该文件夹在nodejs目录下）的路径，环境变量就配置完成了

更改npm镜像

输入 npm config set registry https://registry.npm.taobao.org 来更换到淘宝的镜像，可以用 npm config get registry 来查看当前的镜像(国外的源可能有点慢，所以改成国内的)，这一步做完其实就可以了，因为已经把npm的源改到国内了，cnpm的速度可能会更快一点，但实际使用时如果安装和卸载的时候npm和cnpm万一出现了混用的情况，可能会导致一些error，而且npm的速度搞博客足够了，所以后面只用npm，但是安装cnpm的教程还是写了，想做的可以跟着做。

输入npm install -g cnpm --registry=https://registry.npm.taobao.org来安装cnpm(之后安装模块都能用cnpm来代替npm)

和之前npm修改模块安装路径和缓存路径一样，修改cnpm的，改成和npm的一致

安装完成

nodejs已安装完成，如果安装了cnpm的话，之后用npm命令时可以用cnpm来代替，如 npm install vue -g 用 cnpm install vue -g 来代替，模块就会安装在之前设置的node_modules文件夹下，删除命令就是cnpm uninstall vue -g，因为后面不打算用cnpm，所以后面的安装教程使用npm的，如果要用cnpm，直接用cnpm来代替npm即可。
